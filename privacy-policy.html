Privacy Policy

HoliduTracking Chrome Extension
Last Updated: November 4, 2025
Version: 0.5

TL;DR: The HoliduTracking extension does NOT collect, store, or transmit any personal user data. All data processing happens locally on your device within the Chrome DevTools panel.

1. Introduction

HoliduTracking is a Chrome DevTools Extension (Manifest V3) designed for developers and QA engineers working with Holidu's platform. This privacy policy explains how the extension handles data and respects your privacy.

This extension is developed by Marcel Suetterlin and is intended for use by developers and QA teams to monitor and analyze tracking events from Holidu websites.

2. Data Collection

✅ We Do NOT Collect Any Data

The HoliduTracking extension does NOT collect, store, or transmit any of the following:

  • ❌ Personally identifiable information
  • ❌ Health information
  • ❌ Financial and payment information
  • ❌ Authentication information
  • ❌ Personal communications
  • ❌ Location data
  • ❌ Web browsing history
  • ❌ User activity outside of DevTools
  • ❌ Website content for storage purposes

3. What The Extension Does

The HoliduTracking extension operates entirely within the Chrome DevTools panel. Its functionality includes:

  • Network Request Monitoring: Monitors network requests containing "trackBatch" in the URL when DevTools is open
  • Event Display: Displays tracking events, A/B tests, and tracking data in a user-friendly format within the DevTools panel
  • Local Processing: All data processing happens locally on your device - no data leaves your browser

4. APIs Used

The extension uses Chrome DevTools Extension APIs (implicitly granted by the `devtools_page` declaration):

  • chrome.devtools.panels: Creates the DevTools panel for displaying events
  • chrome.devtools.network: Monitors network requests from the inspected page only
  • chrome.devtools.inspectedWindow: Accesses the current page URL only when user clicks offer links (to preserve URL parameters)

Important: These APIs only work when:

  • You explicitly open Chrome DevTools
  • You navigate to the HoliduTracking panel
  • The extension only sees data from the page you're currently inspecting

5. Data Handling

Network Request Data

  • Only monitored when DevTools is open and HoliduTracking panel is active
  • Only displayed in the DevTools panel - never stored
  • No data is transmitted to any servers
  • All data is cleared when the panel is reloaded
  • Only requests from the currently inspected page are visible (not other tabs or browsing history)

Page URL

  • Only accessed when you click on offer links in impression snapshot events
  • Used only to construct offer URLs that preserve query parameters (checkin, checkout, adults, children)
  • Never stored or transmitted
  • Only accessed from the page currently being inspected

6. Third-Party Services

The HoliduTracking extension does NOT integrate with any third-party services, analytics platforms, or external APIs. It operates completely independently and does not send data to any servers.

7. Data Storage

The extension does NOT use any storage mechanisms:

  • ❌ No localStorage
  • ❌ No sessionStorage
  • ❌ No chrome.storage API
  • ❌ No cookies
  • ❌ No IndexedDB

All event data displayed in the panel is temporary and cleared when:

  • The panel is reloaded
  • DevTools is closed
  • The extension is reloaded

8. User Control

You have complete control over the extension:

  • ✅ Extension only works when you explicitly open DevTools
  • ✅ You can close DevTools at any time to stop monitoring
  • ✅ You can reload the panel to clear all displayed data
  • ✅ You can uninstall the extension at any time
  • ✅ All data processing is transparent and visible in the panel

9. Permissions

The HoliduTracking extension requests NO explicit permissions in its manifest. All functionality is provided through Chrome DevTools Extension APIs, which are automatically granted when you install a DevTools extension.

The extension does NOT request or use:

  • ❌ host_permissions (no external URL access)
  • ❌ tabs API (no tab manipulation)
  • ❌ storage API (no data persistence)
  • ❌ cookies API (no cookie access)
  • ❌ webRequest API (uses DevTools network API instead)
  • ❌ background scripts
  • ❌ content scripts
